# 복습 도장판

1일 · 1주 · 1달 간격으로 강의를 복습하게 도와주는 안드로이드 앱입니다.
홈화면 위젯과 매일 알림이 있습니다.

---

## APK 만드는 법 (설치할 프로그램 없음)

### 1. GitHub 가입
[github.com](https://github.com) → **Sign up** → 이메일·비밀번호·아이디만 정하면 끝. 무료입니다.

### 2. 저장소 만들기
오른쪽 위 **+** → **New repository**

- Repository name: `dojangpan`
- **Public** 을 선택합니다 (Private이면 무료 빌드 시간이 제한됩니다)
- 아래 **Create repository** 를 누릅니다

### 3. 파일 올리기
새로 만들어진 화면에서 **uploading an existing file** 글씨를 누릅니다.

압축을 푼 `dojangpan` 폴더를 **연 다음, 그 안에 있는 것들을 전부 선택해서** 브라우저 화면에 끌어다 놓습니다.
(폴더째 끌지 말고, 폴더 안의 내용물을 끕니다)

> `.github` 폴더가 안 보이면 파일 탐색기에서 **숨김 항목 표시**를 켜세요.
> 이 폴더가 빠지면 APK가 만들어지지 않습니다.

아래 초록색 **Commit changes** 를 누릅니다.

### 4. 기다리기
위쪽 **Actions** 탭을 누르면 `APK 만들기` 가 돌고 있습니다.
노란 점이 초록 체크로 바뀔 때까지 5~10분쯤 걸립니다.

### 5. 폰에 설치
초록 체크가 되면, 저장소 첫 화면 오른쪽 **Releases** 의 **복습 도장판 (최신)** 을 누릅니다.
**폰에서** 그 페이지를 열고 `bokseup-dojangpan.apk` 를 누르면 설치됩니다.

- "출처를 알 수 없는 앱" 경고가 나오면 **허용**을 눌러 주세요. 직접 만든 앱이라 그렇습니다.
- 설치 후 앱을 한 번 열어 **알림 허용**을 눌러 주세요.

### 5-1. 알림이 확실히 울리게 하기 (중요)
앱 → **설정** 을 열면 위쪽에 빨간 안내가 뜹니다. 두 개 다 없어질 때까지 눌러서 처리해 주세요.

- **알림 켜러 가기** — 알림 권한
- **제외하러 가기** — 배터리 절약 제외. **삼성·샤오미 폰은 이걸 안 하면 알림이 자주 씹힙니다.**

마지막으로 **지금 알림 보내보기** 를 눌러 알림이 실제로 뜨는지 확인하세요.

### 6. 위젯 놓기
홈화면 빈 곳을 길게 누르기 → **위젯** → **복습 도장판** 을 홈화면으로 끌어다 놓습니다.

## 처음 할 일

앱을 열고 **설정** 에서

- **1일차 날짜** — 강의를 처음 들은 날
- **전체 강의 수** — 60
- **하루에 듣는 강의 수** — 3
- **시험일** — 있으면 넣고, 없으면 비워 둡니다
- **알림 시각** — 원하는 시각
- **강의 제목 입력** — 강의 목록을 한 줄에 하나씩 붙여넣습니다 (1강부터 순서대로)

## 규칙

하루치 강의를 한 덩어리로 묶고, 그 덩어리를 **다음 날 · 일주일 뒤 · 한 달 뒤** 세 번 복습합니다.

| | 언제 |
|---|---|
| 수강 | N일차 |
| 1일 복습 | N+1일차 |
| 1주 복습 | N+7일차 |
| 1달 복습 | N+30일차 |

밀린 것은 점선 칩으로 계속 따라옵니다. 밀리면 새 강의를 미루고 복습을 먼저 하세요.

### 막혔을 때

복습했는데 잘 안 떠올랐다면 **칩을 길게 누르세요.** 「막힘」으로 표시되고 **3일 뒤 「다시」 줄에 한 번 더** 나옵니다.
「다시」로 나온 것도 또 막히면 다시 길게 눌러 3일 뒤로 미룰 수 있습니다.
도장판에서 그 칸 오른쪽 위에 빨간 점이 붙습니다.

### 며칠 밀렸을 때

화면 위에 **「N일 밀렸습니다 · N일 밀기」** 가 뜹니다. 누르면 남은 일정이 통째로 뒤로 밀려서 오늘부터 다시 맞습니다.
체크한 기록은 그대로 남습니다. 밀린 걸 억지로 따라잡기보다 이 버튼을 쓰는 편이 낫습니다.

### 시험일을 넣으면

- 위젯과 알림에 **D-day** 가 같이 뜹니다
- 일정상 **1달 복습이 시험 뒤로 넘어가는 지점**을 경고해 줍니다

## 고치고 싶을 때

바뀐 파일을 저장소에 올리면 GitHub가 새 APK를 자동으로 만듭니다. 다시 내려받아 덮어 설치하면 기록은 그대로 남습니다.

`keystore.jks` 파일은 **절대 지우지 마세요.** 이 파일로 항상 같은 서명을 하기 때문에 새 APK를 덮어 설치할 수 있습니다. 없어지면 앱을 지웠다 다시 깔아야 하고 기록도 날아갑니다.

## 폰을 바꿀 때

**백업 → 저장 파일 내보내기** 로 `.json` 을 저장해 두고, 새 폰에서 **가져오기** 하면 됩니다.
